# 文案

文案是工厂最重的环节。一条视频的口播稿不只是"一段文字"，它是一个精心设计的结构。

触发方式：`/studio-script`、"写稿"、"写文案"、"口播稿"

## 前置检查

1. 读取 `config/factory.json`，获取定位和风格
2. 确认选题来源：
   - 从飞书选题库读取状态=「已选」的选题
   - 或创作者直接给出选题

## 文案流程

### Step 1：深度研究

围绕选题做补充调研：

```bash
# 搜索相关背景
python3 skill-api/call.py web-search --query "<选题关键词>" --count 10
```

同时检查飞书素材库中是否有相关素材（对标账号做过类似内容）。

### Step 2：输出大纲（必须先过审）

**不要直接写全文。先输出大纲，等创作者确认。**

大纲格式：

```
## 选题：[标题]
## 切入角度：[角度]
## 预估时长：X 分钟

### Hook（前3秒）
[用什么钩住观众]

### 结构
1. [第一段要说什么] — 预估 Xs
2. [第二段要说什么] — 预估 Xs
3. [第三段要说什么] — 预估 Xs

### 核心论据
- [论据1：数据/案例/引用]
- [论据2]
- [论据3]

### 情绪曲线
开头(好奇) → 中间(震撼/共鸣) → 结尾(行动)

### 金句预设
- [结尾金句候选]
```

### Step 3：创作者确认大纲

等创作者确认或调整后再继续。

### Step 4：写口播稿

参考 `知识库/方法论/口播稿框架.md`，按确认的大纲写全文。

**核心要求：**

1. **Hook（前3秒）** — 决定生死。必须有悬念/冲突/反常识
2. **说人话** — 这是说出来的，不是写出来的。短句、口语、不要书面语
3. **每30秒一个钩子** — 防止流失
4. **论据要具体** — "增长了很多"不行，"3个月涨了200万粉"才行
5. **情绪要有起伏** — 不能一直高昂或一直平淡
6. **结尾要有力** — 金句收尾

**格式：**

```
[画面指引] 开头画面描述

正文第一段...

[画面指引] 切换画面描述

正文第二段...

[画面指引] 结尾画面

金句收尾。
```

### Step 5：存入飞书

**1. 创建飞书云文档（正文）：**

```bash
lark-cli docs +create --as user --title "[选题标题] - 口播稿" --markdown "<script_content>"
```

**2. 写入飞书脚本库（索引记录）：**

```bash
lark-cli base +record-upsert --as user \
  --base-token <base_token> --table-id <script_table_id> \
  --json '{
    "fields": {
      "标题": "脚本标题",
      "关联选题": ["<topic_record_id>"],
      "状态": "初稿",
      "文档": {"text": "口播稿", "link": "<doc_url>"},
      "审核意见": ""
    }
  }'
```

**3. 更新选题状态：**

```bash
lark-cli base +record-upsert --as user \
  --base-token <base_token> --table-id <topic_table_id> \
  --record-id <record_id> \
  --json '{"fields": {"状态": "写稿中"}}'
```

### Step 6：打磨循环

文案很少一次就好：

```
初稿 → 创作者反馈 → 修改 → 再反馈 → ... → 定稿
```

每次修改都更新飞书文档（`lark-cli docs +update --mode overwrite`），保持单一来源。
定稿后更新脚本库状态为「终稿」。
