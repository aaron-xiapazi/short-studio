# 选题

选题不是"找热点"，是**在你的定位下找到适合你讲的热点**。

触发方式：`/studio-topic`、"选题"、"找选题"、"今天做什么"

## 前置检查

1. 读取 `config/factory.json`，获取创作者定位
2. 如果不存在，提示先运行 `/studio-init`

## 选题流程

### Step 1：抓取热点

从多渠道获取当日热点（调用 agent-reach 或 web-access skill）：
- 微博热搜
- 知乎热榜
- 36kr
- 抖音热点
- 与定位相关的垂直渠道

### Step 2：定位匹配筛选

用创作者定位过滤热点。每个候选选题需要回答：

1. **跟我的定位有关吗？** — 不相关的直接淘汰
2. **我有独特视角吗？** — 我能说出别人说不了的东西吗
3. **受众关心吗？** — 目标受众会停下来看吗
4. **我有素材/经验支撑吗？** — 能说出干货还是只能泛泛而谈

### Step 3：选题评估

参考 `知识库/方法论/选题评估矩阵.md`，从四个维度打分（1-10）：

| 维度 | 说明 |
|------|------|
| 热度 | 当前讨论度，时效性 |
| 争议 | 有没有可以站队的观点 |
| 关联 | 跟创作者定位的契合度 |
| 独特 | 能不能讲出不一样的角度 |

### Step 4：输出选题卡

参考 `templates/topic-brief.md` 模板，输出 3-5 个选题：

```
## 选题 1：[标题]
- 切入角度：[怎么讲]
- 为什么适合你讲：[定位匹配点]
- 热度：X/10 | 争议：X/10 | 关联：X/10 | 独特：X/10
- 来源：[哪里看到的]
```

### Step 5：写入多维表格

用户选定后，写入飞书选题库：

```bash
lark-cli base +record-create --as user \
  --app-token <topic_base_token> --table-id <topic_table_id> \
  --data '{"fields":{"标题":"<title>","角度":"<angle>","来源":"<source>","热度":<score>,"状态":"已选","备注":"<analysis>"}}'
```

### Step 6：参考历史（如有数据）

如果 `知识库/自有库/published.jsonl` 存在：
- 检查类似选题过去的表现
- 避免重复做过的选题
- 参考效果好的选题方向
