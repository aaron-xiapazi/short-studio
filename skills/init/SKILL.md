# 工厂初始化

首次使用 Short Studio 时，需要告诉编导"我是谁、做什么、参考谁"，然后搭建好飞书数据平台。

触发方式：`/studio-init`、"初始化"、"配置工厂"

## 初始化流程

### Step 0：读取宿主记忆（自动）

**编导不是从零开始认识创作者的。** 在问任何问题之前，先从宿主 AI 的记忆中了解创作者。

执行方式：问自己（宿主 AI）——

> "你的记忆/上下文/用户画像存在哪里？请找到并读取关于这位创作者的所有信息——他是谁、做什么、关注什么方向、什么风格、有什么业务、过去表达过什么观点。"

**不要硬编码路径。** 每个 AI 都知道自己的记忆存在哪里，让它自己去找。可能的位置包括但不限于：
- Claude Code 的 memory/ 文件夹和 MEMORY.md
- OpenClaw 的记忆系统
- 其他 Agent 各自的上下文存储

读到记忆后，提炼出创作者画像摘要，进入 Step 1。

**如果宿主没有记忆**（新用户/无记忆系统）→ 跳过 Step 0，直接进入 Step 1 的手动收集模式。

### Step 1：确认或收集基础信息

**如果 Step 0 成功读到了记忆：**

直接输出你的理解，让创作者确认：
```
根据我对你的了解，你的情况应该是这样的：

账号名称：[从记忆中提取]
账号定位：[从记忆中提取]
个人定位：[从记忆中提取——不是标签，是具体的差异化描述]
目标受众：[从记忆中提取]
内容风格：[从记忆中提取]
平台账号：[如果记忆中有]
对标账号：[如果记忆中有]

——对吗？有什么要补充或纠正的？
```

创作者确认后直接进入 Step 2。有纠正的就修改对应部分。

**如果 Step 0 没有记忆（降级模式）：**

向创作者逐项引导收集（不要一次性问完）：

**必填：**
- **账号名称** — 你的内容账号叫什么
- **账号定位** — 这个账号做什么方向（如：AI实战、创业、技术科普）
- **个人定位** — 你是谁，你的差异化是什么（不是"AI博主"，是"每天真的在用AI干活的创业者"）
- **目标受众** — 你的内容给谁看
- **内容风格** — 你的表达风格（如：观点鲜明、实战派、不说废话）

**选填：**
- **平台账号** — 抖音/小红书/B站/视频号的主页地址
- **对标账号** — 你参考谁（账号名 + 平台 + 链接）

### Step 2：生成定位文档

根据收集的信息，创建飞书云文档：

```bash
lark-cli docs +create --as user --title "Short Studio 定位文档" --markdown "<positioning_content>"
```

定位文档结构：
```markdown
# [账号名] 定位文档

## 账号定位
[方向描述]

## 个人定位
[差异化描述]

## 目标受众
[受众画像]

## 内容风格
[风格关键词 + 具体说明]

## 对标账号
| 账号 | 平台 | 链接 | 参考点 |
|------|------|------|--------|

## 平台账号
| 平台 | 链接 |
|------|------|

---
*生成时间：YYYY-MM-DD*
*这是活文档，会随着创作持续更新*
```

### Step 3：创建飞书 Base（一个 Base，三张表）

创建内容工厂的数据库——一个 Base 包含选题库、脚本库、素材库。

**在执行前，先读 lark-base skill 的字段属性规范。**

```bash
# 1. 创建 Base
lark-cli base +base-create --as user --name "Studio 内容工厂"
# 记录 base_token，默认会有一张表

# 2. 把默认表改名为选题库，创建字段
# 字段：标题(默认已有)、角度(文本)、状态(单选)、来源(文本)、备注(文本)

# 3. 创建脚本库表
lark-cli base +table-create --as user --base-token <token> --name "脚本库"
# 字段：标题(文本)、关联选题(关联→选题库)、状态(单选: 大纲/初稿/终稿)、
#       文档(超链接)、审核意见(文本)

# 4. 创建素材库表
lark-cli base +table-create --as user --base-token <token> --name "素材库"
# 字段：标题(文本)、作者(文本)、平台(单选: 抖音/小红书/B站)、
#       链接(超链接)、转写文案(文本)、播放量(数字)、点赞量(数字)、采集时间(日期)
```

**选题库字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | 文本 | 选题标题 |
| 角度 | 文本 | 切入点 |
| 状态 | 单选 | 待选 / 已选 / 写稿中 / 待审 / 已发布 |
| 来源 | 文本 | 热点链接、参考视频等 |
| 备注 | 文本 | |

**不要加 AI 评分字段。** AI 的判断在对话里说，表里只存真实数据和状态。

### Step 4：关联飞书机器人

引导创作者关联飞书机器人，建立编导和创作者的对话通道：

1. 如果创作者已有飞书机器人 → 获取 bot_id
2. 如果没有 → 引导创建或使用我们提供的机器人
3. 创作者添加机器人为联系人，或拉入群聊
4. 记录 chat_id（对话/群聊 ID）

关联后，编导可以通过飞书 IM 主动推送消息：
```bash
lark-cli im +messages-send --chat-id <chat_id> --msg-type text --content '{"text":"编导已上线，随时待命！"}'
```

### Step 5：写入配置 + 确认

将所有配置写入 `config/factory.json`，输出摘要：

```
✅ 工厂初始化完成

定位：[一句话总结]
定位文档：[飞书文档链接]
数据库：[飞书 Base 链接]
  - 选题库 ✅
  - 脚本库 ✅
  - 素材库 ✅
飞书机器人：[已关联 / 待关联]

现在可以开始了：
- 跟编导聊天，说你想干嘛
- /studio-topic  找选题
- /studio-scrape 采集对标账号内容
```

## 更新定位

如果 `config/factory.json` 已存在，进入更新模式：
- 显示当前定位
- 问创作者要更新哪部分
- 只更新指定部分，不重新走全流程

## 权限

| 操作 | scope |
|------|-------|
| 创建文档 | `docx:document`, `drive:drive` |
| 创建多维表格 | `bitable:app` |
| 发送消息 | `im:message` |
